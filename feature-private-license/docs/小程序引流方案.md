# 小程序引流方案（AI Agent Platform ↔ 主站 H5 变现闭环）

> 目标：用微信小程序作为**免费流量入口**，把用户导到主站 H5 `/pricing` 完成会员/企业版购买，
> **复用现有微信支付**，不在小程序内另起支付。小程序本身已通过 CloudBase「小程序成长计划」
> 提供免费模型额度（见 `client/src/config/aibakModels.ts`），是天然引流载体。

---

## 一、引流闭环（已在本地验证）

```
微信小程序（免费聊天/试用）
   │  用户看到「升级会员 / 私有化部署」入口
   ▼
web-view 打开 H5：https://aibak.site/pricing
   │  展示 免费/专业¥9.9/旗舰¥19.9/团队¥99 + 企业版¥299/¥599/¥999
   ▼
点击「立即购买」→ 未登录先走注册/登录（微信扫码或邮箱）
   ▼
下单（复用现有微信支付网关，真实打通）
   ▼
支付成功 → webhook 履约
   ├─ 订阅/积分包 → 激活（credit-ledger）
   └─ 企业版(private_license) → 调金网通签发 License → 用户控制台下载
```

**关键验证结论（2026-07-18 本地实测）**：
- 金网通 store 模式 B 签发接口 `POST /api/admin/issue`（version/validDays/seats/email/orderNo）已实测返回合法 license（sign 正确），契约与 `private-license.service.ts` 完全对齐。
- 公开接口 `GET /api/billing/private-license-packages` **无需登录**即可拉取企业版套餐（billing.ts 第96行无 `requireAuth`），可作为小程序展示数据源。
- 下单接口 `POST /private-license/order` 需登录，故小程序引流必须在 H5 内引导登录后再购买（符合合规与风控）。

## 二、小程序端落地（web-view 方案，零后端改动）

小程序只需一个 `web-view` 页，把 H5 包进去，**不需要任何新后端接口**。

### 1. 小程序页面：`pages/pricing/pricing.wxml`
```xml
<web-view src="{{url}}"></web-view>
```

### 2. `pages/pricing/pricing.js`
```js
Page({
  data: { url: '' },
  onLoad(query) {
    // 把小程序 openid/渠道参数透传给 H5，便于归因（可选）
    const channel = 'miniprogram';
    this.setData({
      url: `https://aibak.site/pricing?from=${channel}&utm_source=miniprogram`
    });
  }
})
```

### 3. 小程序 `app.json` 注册页面 + tabBar 入口
```json
{
  "pages": ["pages/index/index", "pages/pricing/pricing"],
  "tabBar": {
    "list": [
      { "pagePath": "pages/index/index", "text": "免费试用" },
      { "pagePath": "pages/pricing/pricing", "text": "升级会员" }
    ]
  }
}
```

### 4. 业务域名配置（微信公众平台必做）
- 登录微信小程序后台 → 开发 → 开发管理 → 业务域名
- 添加 `aibak.site`（需下载校验文件放到主站根路径，nginx 已具备放行能力）
- 否则 `web-view` 无法加载主站 H5。

## 三、H5 侧配合（已完成，无需新开发）
- `/pricing` 路由已存在（`client/src/router.tsx` 第75行），企业版卡片已在 `PricingPage.tsx` 渲染。
- 企业版按钮「立即购买授权」→ `createPrivateLicenseOrder` → 微信支付 → webhook 自动签发。
- 引流归因：H5 读取 `?from=miniprogram` 仅作统计，不阻断流程。

---

## 四、为什么不复用金网通小程序
金网通 store 是**独立产品**（企业局域网互联），其购买页 `https://aibak.site/jinwangtong/` 已有手动模式。
本方案让 AI Agent Platform 主站统一收口支付与会员体系，金网通仅作**私有化 License 签发后端**，
避免两套支付/账号体系割裂，符合「全复用现有 billing」原则。
