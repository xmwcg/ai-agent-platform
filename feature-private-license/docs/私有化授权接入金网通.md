# 私有化授权 · 复用金网通 License 接入说明

> 目标：商城"企业版/私有化"套餐下单后，自动调金网通 license 签发，客户下载 `license.json`。
> 现状：金网通已部署 `/opt/jinwangtong/store`（Node :3100），HMAC-SHA256 机制就绪（memory ID:37639690）。

---

## 一、复用方式（零新增代码）
商城私有化订单支付成功后，复用金网通管理接口签发 license：

```js
// 在 order.service.js 的 handleNotify 中，对 planId === 'ent' 走私有化签发
async function issuePrivateLicense(userEmail, version) {
  const resp = await axios.post('http://127.0.0.1:3100/api/admin/issue', {
    email: userEmail,
    product: 'ai-gateway',
    version,                 // standard / pro / flagship
  }, { headers: { Authorization: `Bearer ${process.env.JINWANGTONG_ADMIN_TOKEN}` } });
  return resp.data.license; // license.json 内容
}
```

- 密钥一致性：金网通 `store/license-issuer.js` 与客户端 `scripts/license.ps1` 的 HMAC 密钥必须一致（默认 `JinWangTong-2026-Local-Key`）。改密钥需两端同步。
- 管理 token：`/opt/jinwangtong/store/.env` 的 `ADMIN_TOKEN`。

## 二、nginx 复用
金网通购买页已在 `https://aibak.site/jinwangtong/`，商城"私有化"页直接内嵌该购买流程或跳转，无需新建页面。

## 四、部署步骤
1. 商城 `planId='ent'` 订单回调里调用 `issuePrivateLicense`。
2. 返回的 `license.json` 存到商城数据库，用户控制台可下载。
3. 客户端用 `scripts/license.ps1` 校验（复用金网通已发布逻辑）。

## 五、注意事项
- 不改动金网通代码，仅商城侧调用其 admin 签发接口。
- `ADMIN_TOKEN` 通过环境变量从 `.env` 注入商城容器，不硬编码。
- 私有化交付包 = New API 部署配置 + 本方案文档 + license。
