## 三、问题②：采购 token vs 大厂月付会员，成本怎么控？

**答：你项目已经是标准答案，不需要"买大厂月付会员"。**

你的成本结构（来自 `cost-control.service.ts` + `billing.ts`）：
- **模型成本基准**：DeepSeek 混合价 输入¥1/百万 token、输出¥2/百万 token（`estimateCostFen`）。
- **自家 Key 直连** 比 **中转站月付会员** 更便宜：你已直连 DeepSeek/智谱/通义/豆包官方，单价就是厂商价，无中转加价。
- **两道成本闸门**（已有的，直接用）：
  - 第一道：`subscription.ts` 日配额上限（次数锁）
  - 第二道：`checkCostValve` 日成本预算（钱锁），free 50分/pro 500分/日，超限即限流
- **重成本资源按次扣积分**（`PER_USE_COST`）：文生图¥0.2、视频¥2、开放API¥0.1——用户实付成本+加价，平台不垫付。

**采购决策建议**：
- 不要买"大厂月付会员"（GPT Plus/Claude Pro 之类），那是 C 端订阅，不能用于 API 转售，且违规。
- 用 **API 按量计费**（你已在做）：DeepSeek/硅基流动最便宜，硅基流动注册送额度可零成本冷启动。
- 若想再加一层稳定性，部署 **New API 网关**聚合硅基流动+火山+DeepSeek，做渠道优选，但**额度仍走你现有积分账本**。

## 四、问题③：用户买你会员，利润如何体现？方案有设计吗？

**答：有，且就在你代码里——无需我新设计，只需把账算清给你看。**

利润模型（基于 `billing.ts` + `cost-control.service.ts`）：

| 套餐 | 月费 | 日成本预算(封顶) | 按 DeepSeek 价实际最大日成本 | 月毛利(按用满) |
|------|------|------------------|------------------------------|----------------|
| 专业 ¥9.9 | ¥9.9 | ¥5.0/日 | ≈¥5（用满才到） | ¥9.9 − ¥5×30≈ **¥-140?** ⚠️ |

⚠️ **注意**：上表"用满"是极端假设。实际多数用户用不满日配额。但暴露一个真实风险：
- 专业版 ¥9.9/月，若用户**每天用满 500 条对话**（≈750k token/日 ≈ ¥1.5/日成本），月成本 ≈¥45 > ¥9.9 → **亏损**。
- 你的利润靠两点兜底：① 大部分用户用不满；② `PLAN_AI_BUDGET_FEN` 已把 pro 日成本锁在 ¥5（约 2200 条/日），实际 500 条配额远用不满预算；③ 重资源按次扣积分不垫付。

**真实利润测算（典型用户，非用满）**：
- 专业版用户日均 50 条对话 ≈ ¥0.15/日成本 → 月成本 ¥4.5，月费 ¥9.9 → **毛利 ≈ ¥5.4（55%）**
- 旗舰版 ¥19.9，日预算无限但受配额，重度用户月成本约 ¥10-20 → **毛利≈0~¥10**（旗舰靠"无限"溢价+BYOK 引导）
- 积分包：100积分¥9.9（成本约¥2-5）→ **毛利 50%+**

**利润体现位置（你代码里已有）**：
- 会员费进账：`webhook.routes.ts` → `activateSubscription`
- 实际成本计量：`recordAiCost` → Redis 日成本
- 利润 = 会员费 − 累计 `getDailyCost` 汇总（需加一个"毛利看板"，目前代码有计量但**没有聚合报表**）

**建议补的（增量，非新建）**：在现有 Admin/控制台加一个"毛利看板"，按月汇总 `∑会员费 − ∑recordAiCost − ∑积分成本`，这是你目前唯一缺的"利润可视化"环节。

## 五、推荐整合架构（不冲突版）

```
短视频引流(视频号/抖音/小红书)
        ↓
小程序 / PricingPage(你已有) ──复用──> /api/billing/orders(你已有)
        ↓ 微信支付(你已打通, wx4215...)
        ↓ webhook.routes.ts(你已有, 验签+幂等)
   积分账本 credit-ledger(你已有) + 订阅激活(你已有)
        ↓
   模型调用 ai-gateway(你已有, 直连 DeepSeek等)
        ↓ estimateCostFen + 日成本阀门(你已有, 锁成本)
   利润 = 会员费 − 实际成本(需加毛利看板)
        ↘ 私有化: PricingPage 加"企业版"入口 → 金网通 License(你已有)
```

**只新增、不重复**：
1. New API 网关作为**可选后端**（优化稳定性，额度仍走你积分账本）
2. PricingPage 加"私有化/企业版"卡片 → 复用支付发 License
3. 小程序（新渠道载体）→ 调你现有 orders API
4. 毛利看板（Admin 增量）
5. 短视频内容（纯运营）
