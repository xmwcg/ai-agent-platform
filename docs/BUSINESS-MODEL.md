# 商业变现方案（Monetization Blueprint）

> 本文档定义 Reasonix AI Agent 平台的商业闭环：变现路径、套餐设计、付费方式、增长与合规。
> 定位：**对标 Dify / Coze / FastGPT / LibreChat，面向中小企业与个人创作者的国产大模型驱动平台**。

---

## 一、变现途径总览

| 途径 | 形态 | 说明 | 状态 |
|------|------|------|------|
| 1. 订阅套餐 | SaaS 月/年费 | 免费/专业/旗舰三档，按配额限流 | ✅ 已实现 |
| 2. 按量计费 | 积分（credits） | 高成本操作（文生视频/长方案）扣积分 | 🟢 已埋点 |
| 3. API 开放平台 | 开发者付费 | 对外提供对话/RAG/翻译/生成 API | 🟡 路由已就绪 |
| 4. 企业私有化 | 一次性授权 + 年维保 | 部署到客户内网，含定制模型 | 📋 路线图 |
| 5. 增值服务 | 知识库托管 / 智能客服坐席 | 按席位/文档量计费 | 📋 路线图 |
| 6. 渠道分润 | 模型厂商返佣 | 用户通过平台消耗第三方模型获得返点 | 📋 路线图 |

## 二、套餐与配额设计（已实现）

三档套餐让「免费体验 → 付费转化 → 旗舰无限制」形成漏斗：

- **免费版**：每日配额（对话 20 / 翻译 10 / 转换 5 / 方案 3 / 媒体 2 / 客服 20），适合体验与口碑传播。
- **专业版**：39 元/月，配额提升 10-25 倍，无限 RAG，5 个 MCP 服务器，**主推转化档**。
- **旗舰版**：99 元/月，无限配额 + 20 个模型配置 + 专属客服，面向团队与企业。

> 配额闸门：中间件 `enforceQuota(resource)` 按 Redis 日维度计数，超阈值返回 `402` 并引导升级，自然形成付费转化点。

## 三、付费方式（已实现抽象层）

`services/payment.service.ts` 定义 `PaymentGateway` 接口，已实现：

- **Mock 支付**：`DEFAULT_PAY_PROVIDER=mock`，一键跑通「下单 → 支付 → 激活会员」，用于演示与联调。
- **微信支付 v3**：实现 `WechatGateway`，含统一下单、回调验签骨架（补证书即生产可用）。
- **Stripe**：实现 `StripeGateway`，支持国际卡支付。

新增渠道只需实现接口并在工厂 `getPaymentGateway` 注册，业务层零改动。

## 四、转化与增长策略

1. **配额即营销**：免费用户触达上限时强制曝光升级页（`/pricing`），转化路径最短。
2. **模型配置引流**：免费版仅允许 1 个自定义模型，专业版 3 个、旗舰版 20 个——直接拉动升级。
3. **智能客服嵌出**：用户把客服挂到自己网站，带来自然流量与外链，形成 B 端获客。
4. **内容生产裂变**：文生图/视频产出可分享至自媒体，间接传播品牌。
5. **企业版锚定**：公开旗舰版价格，向上销售私有化与定制。

---

## 五、变现合规要点

- 价格以「分」存储，避免浮点误差；展示层统一除 100。
- 支付回调必须验签（微信 v3 证书 / Stripe webhook secret），防止伪造订单。
- 会员过期自动降级 free（`resolveUserPlan` 已实现），无脏数据。
- 用户数据（apiKey 等）落库需加密，生产环境建议 KMS/字段级加密。
- 提供订单历史（`GET /billing/orders/history`）与取消订阅（`POST /billing/subscription/cancel`），满足消费者权益。
