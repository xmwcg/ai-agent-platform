# 中转站模块说明（已整合进平台）

中转站不再是独立网站，而是 **ai-agent-platform 平台内的一个模块**，跟随平台一起部署到 www.aibak.site，对外暴露地址为：

```
https://www.aibak.site/api/relay/v1
```

金网通（企业局域网管控）控制台⑥选「我的中转站·智控版」，填上面的地址 + 下面发放的令牌即可。

## 它做了什么（MVP）
- **接上游**：平台侧配置多家大模型渠道（DeepSeek / 通义 / OpenAI 等），密钥只存平台。
- **发令牌**：每个客户 License 发一把中转站令牌（`sk-relay-xxx`），绑定额度。
- **按量记账**：每次调用扣减额度、留用量流水，超额拦截。
- **OpenAI 兼容**：对外就是标准 `/v1/chat/completions` 和 `/v1/models`，金网通零改对接。

## 管理后台（管理员用）
浏览器打开：`https://www.aibak.site/api/relay/admin`
1. 先填你的管理员 JWT（从平台登录后获取）。
2. **上游渠道**：填各家模型的 baseURL + apiKey，可多个（自动按权重分摊）。
3. **客户令牌**：填 LicenseId + 配额，点「发放令牌」——**明文只显示一次**，复制给对应金网通客户。
4. **用量统计**：按 LicenseId 看累计用量与调用次数。

## 接口一览
| 方法 + 路径 | 说明 |
|----|----|
| `POST /api/relay/v1/chat/completions` | 公开（凭客户令牌）：转发到上游，记账 |
| `GET  /api/relay/v1/models` | 公开（凭客户令牌）：列出可用模型 |
| `GET  /api/relay/admin` | 管理员：HTML 管理页 |
| `GET/POST/DELETE /api/relay/admin/channels` | 管理员：渠道增删查 |
| `GET/POST /api/relay/admin/tokens` | 管理员：令牌发放/列表 |
| `POST /api/relay/admin/tokens/:id/revoke` | 管理员：吊销令牌 |
| `GET  /api/relay/admin/usage` | 管理员：用量统计 |

## 金网通侧怎么填
- 厂商：我的中转站·智控版
- BaseURL：`https://www.aibak.site/api/relay/v1`
- 鉴权密钥：上面发放的 `sk-relay-xxx` 令牌

## 部署
无需改 `docker-compose.yml`。中转站是 `server/` 里的新代码，平台重新构建部署（你现有的 CI/CD 或 `docker compose up -d --build`）即自动上线。
