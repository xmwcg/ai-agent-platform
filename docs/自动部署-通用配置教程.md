# 自动部署配置与通用化教程

> 一句话目标：**在任意一台电脑（甚至手机）改完代码，只要提交，服务器约 1 分钟内自动更新上线，全程零手动运维。**

本文说明这套自动部署的原理、机制，以及**如何把项目拷贝/克隆到任何新电脑后仍然照常工作**（通用化配置）。

---

## 一、整体原理（30 秒看懂）

```
┌──────────────┐   git commit    ┌─────────────────────┐   每分钟轮询    ┌──────────────┐
│  你的电脑     │ ─(钩子自动push)→ │  cnb.cool 仓库       │ ←──(拉模式)──── │  服务器       │
│ (本地开发)    │                 │ aibak.site/...(main) │                │ 159.75.124.59 │
└──────────────┘                 │   = 部署真源         │                │  自动构建部署  │
        │                        └─────────┬───────────┘                └──────────────┘
   手机网页直接改 ───────────────────────────┘（同样进入部署真源）
                                            │ .cnb.yml 自动镜像
                                            ▼
                                    GitHub (仅备份 + 跑 CI 测试)
```

**三个角色：**
- **cnb.cool 仓库** = 唯一「部署真源」。任何进入它 main 分支的代码都会被部署。
- **服务器** = 「拉模式」部署方。它主动每分钟去 cnb.cool 看一眼，有变化就拉下来构建、重启容器。
- **GitHub** = 退居二线，只做镜像备份 + 跑自动化测试（CI），**不再负责部署**。

**为什么用「服务器主动拉」而不是「CI 推服务器」？**
腾讯云安全组会拦截 CI 的临时入站 IP，导致「推」模式不稳定。改成服务器纯出站轮询拉取，绕开入站限制，稳定可靠。代价是最多约 1 分钟延迟。

## 二、完整链路机制（逐环节）

1. **本地 commit → 自动推送**
   本仓库装了一个 `post-commit` 钩子：你在 `main` 分支每次 `git commit` 后，它自动执行 `git push cnb main`。你**不需要手动 push**。

2. **cnb.cool 收到代码 → 自动镜像到 GitHub**
   仓库根的 `.cnb.yml` 定义：push main 时用令牌把代码 `--force` 推到 GitHub（`sync-to-github`），触发 GitHub Actions 跑测试。另有每 4 小时反向从 GitHub 合并回来（`sync-from-github`），双向兜底。

3. **服务器每分钟轮询 → 检测变化 → 部署**
   服务器上 cron 每分钟执行 `deploy/cnb-watcher.sh`：
   - `git ls-remote` 取 cnb.cool 的最新 SHA；
   - 与本地状态文件 `/opt/.cnb-deploy-sha` 比对；
   - **有变化**才 `fetch + reset --hard`，然后后台执行 `deploy/push-deploy.sh`（tsc/vite 构建 + `docker compose` 重建容器）；
   - 无变化则零成本退出。
   > 该脚本**只硬编码盯 `aibak.site/ai-agent-platform` 这一个仓库**，其他 cnb 项目变化不会触发它。

4. **手机远程开发**
   直接在 cnb.cool 网页编辑并提交，等价于第 2 步，服务器照常自动部署。**彻底绕开 GitHub 断网问题。**

## 三、为什么「直接拷贝到新电脑」会失效？

这套机制里，有三样东西**不随 `git clone`/拷贝自动带过去**：

| # | 失效点 | 原因 |
|---|--------|------|
| 1 | `.git/hooks/post-commit`（自动推送钩子） | 位于 `.git/` 目录，**从不纳入版本控制**，clone 不会带上 → commit 不再自动推送 |
| 2 | `cnb` remote | 新电脑 clone 默认只有 `origin`，没有 `cnb` → 无处可推 |
| 3 | cnb.cool 推送凭据 | 凭据存在本机凭据管理器，新电脑需重新登录一次 |

> 服务器端（watcher/cron）和 `.cnb.yml` 是**一次性/可移植**的，换电脑无需重做。

**本项目已做通用化改造，解决上述问题** —— 见下一节。

---

## 四、通用化方案（换任意电脑一键搞定）

核心思路：**把钩子放进仓库（受版本控制），用一个一键脚本在新电脑上启用它。**

已加入仓库的文件：

```
scripts/git-hooks/post-commit      # 通用钩子（随仓库分发，clone 就带上）
scripts/setup-auto-deploy.ps1      # 一键配置（Windows / PowerShell）
scripts/setup-auto-deploy.sh       # 一键配置（macOS / Linux）
```

一键脚本做三件事（幂等，可重复运行）：
1. `git config core.hooksPath scripts/git-hooks` —— 让 git 改用**仓库内**的钩子目录（这样钩子内容也随 `git pull` 更新）；
2. 确保 `cnb` remote 存在（没有则自动 `git remote add`）；
3. 验证 cnb 凭据是否可用，并给出登录指引。

> 说明：`core.hooksPath` 是每个仓库的本地设置，`git clone` 不会自动带过来——这是 Git 的安全设计（防止 clone 别人仓库就自动执行代码）。所以**新电脑必须运行一次这个脚本**，这是无法也不应省略的一步。

## 五、在新电脑上启用（操作步骤）

### Windows

```powershell
# 1) 克隆项目（或直接拷贝整个项目文件夹，含 .git）
git clone https://cnb.cool/aibak.site/ai-agent-platform.git
cd ai-agent-platform

# 2) 一键配置（只需运行一次）
powershell -ExecutionPolicy Bypass -File scripts\setup-auto-deploy.ps1
#   若网络慢卡在验证步骤，可跳过验证（不影响配置）：
#   $env:SKIP_VERIFY="1"; powershell -ExecutionPolicy Bypass -File scripts\setup-auto-deploy.ps1

# 3) 首次可能需登录一次 cnb 凭据（之后长期免密）
git push cnb main
```

### macOS / Linux

```bash
git clone https://cnb.cool/aibak.site/ai-agent-platform.git
cd ai-agent-platform
bash scripts/setup-auto-deploy.sh      # 只需运行一次
git push cnb main                       # 首次登录一次凭据
```

完成后，**在 `main` 分支每次 `git commit` 都会自动推送并触发部署**。

## 六、日常使用（配置完成后）

- **电脑开发**：改代码 → `git commit`（钩子自动 push cnb）→ 等约 1 分钟自动上线。
- **手机开发**：在 cnb.cool 网页改代码并提交 → 自动上线。
- 无需任何手动 `git push`、无需登服务器。

## 七、可配置项与排错

**改部署 remote 名 / 分支**（默认 `cnb` / `main`）：
```bash
git config deploy.remote <remoteName>
git config deploy.branch <branchName>
```

**常见问题**：

| 现象 | 原因 / 处理 |
|------|-------------|
| commit 后没有自动推送 | 未运行 setup 脚本（`core.hooksPath` 未设）；或不在 `main` 分支；或没有 `cnb` remote |
| 推送报认证失败 | 首次需登录：`git push cnb main`，输入 cnb.cool 用户名 + 访问令牌(Token) |
| 推了但网站没更新 | 服务器 watcher 每分钟一轮，稍等；或登服务器看 `/var/log/cnb-watcher.log`、`/var/log/ai-platform-deploy.log` |
| Windows 钩子没执行 | 需安装 Git for Windows（自带 bash）；`core.hooksPath` 下的脚本由 git 用内置 sh 执行 |
| 想临时不自动推送 | `git config core.hooksPath ""` 关闭；恢复再运行 setup 脚本 |

**服务器侧一次性配置（已就绪，仅备查）**：
```
# /etc/cron.d/cnb-watcher
* * * * * root /bin/bash /opt/ai-agent-platform/deploy/cnb-watcher.sh
```

## 八、安全与边界

- 钩子仅在 `main` 分支触发，其他分支 commit 不会误上线。
- watcher 只盯本项目仓库，不影响服务器上其他 cnb 项目。
- 钩子非阻塞：推送失败只提示、不回滚你的本地 commit。
- 半成品保护：请只提交完整可运行的改动；未完成代码不要 commit 到 main，否则会被推上线导致服务异常。
