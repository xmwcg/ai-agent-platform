# 痛点调研与差异化模块设计（Reasonix AI）

> 本章基于 2026 年全网关于「AI / 大模型 / 知识库 / AI 客服 / AI 自动化」的需求与痛点调研，
> 提炼出本平台对应的**差异化模块**，避免与 Dify / n8n / LobeChat / FastGPT 等同质化。

## 一、调研结论：用户最痛的 6 件事

| # | 痛点 | 典型声音 | 本平台解法 |
|---|------|----------|-----------|
| 1 | **部署/配置太复杂** | "Docker 一堆环境变量，Redis/Mongo 起不来就白干" | `GET /api/diagnostics` 一键自检 + 前端「部署自检」页，逐项给修复建议；Redis 缺失自动降级内存实现，0 依赖可跑 |
| 2 | **AI 客服"黑盒、不可信"** | "它编的答案我不敢用，也不知道依据哪篇文档" | RAG **答案可追溯**（返回来源 docId/标题/置信度）+ 触发词**转人工** + **满意度评分**闭环 |
| 3 | **上手门槛高、不知从哪开始** | "功能很多，但搭一个客服要建文档、配模型、写提示词" | **场景化快速启动模板**（客服/新媒体/备考/研发），一键生成「知识文档+机器人」 |
| 4 | **多模型接入割裂** | "每个平台只接自己的模型，换厂商要改代码" | **大模型配置中心**运行时自助接入 OpenAI/DeepSeek/混元/通义/智谱等，全平台统一驱动 |
| 5 | **媒体生产厂商分散、接入难** | "文生视频这个厂商、图生图那个厂商，密钥各管各的" | **媒体生成 Provider 抽象**：混元/可灵/即梦/Mock 统一接口，配置即切换，无密钥走演示 |
| 6 | **企业协作与开放变现缺位** | "团队怎么管权限？我的能力怎么对外开放收费？" | **团队 RBAC** + **开放 API 市场按量计费**（API Key 签发、日配额计量、超阈 429） |

## 二、对标项目的"借鉴而非抄袭"差异化定位

- **Dify**：工作流 + RAG + 模型管理 —— 我们取其「模型管理」内核，但强化**国内主模型一键接入**与**商业变现闭环**。
- **n8n**：节点化自动化 —— 我们借鉴其**模板市场**思路，落地为「场景化快速启动模板」，降低非技术用户门槛。
- **LobeChat**：多模态对话 UI —— 我们借鉴其**信息架构**（分组侧边栏），并延伸到学习/创作/生产工具矩阵。
- **FastGPT**：知识库客服 —— 我们补上其薄弱环节：**答案可追溯 + 转人工 + 满意度**，让客服"可信、可追责"。
- **LibreChat**：多模型 Artifacts —— 我们取其「统一对话层」，由模型配置中心驱动。

**本平台独有亮点（避免同质化）**：
1. 答案可追溯的智能客服（来源引用 + 转人工 + 满意度）。
2. 场景化一键启动模板（对标模板市场，但聚焦"知识库+客服"组合）。
3. 部署自检看板（降低安装痛点的硬科技）。
4. 开放 API 市场按量计费（把能力变成可收费的开放平台）。
5. 媒体生成多厂商抽象（混元/可灵/即梦统一调度）。

## 四、生产化建议（下一阶段）

1. **支付真实验签联调**：当前已落地 Stripe HMAC-SHA256 与微信 AES-256-GCM 解密 + RSA 验签逻辑，需配置 `STRIPE_WEBHOOK_SECRET` / `WECHAT_PLATFORM_CERT` 与平台证书公钥做端到端联调。
2. **团队版 RBAC 深化**：资源级授权（知识库/客服按团队隔离）、邀请链接、审计日志。
3. **开放 API 市场计费**：按 `usedToday` 对接套餐额度与积分扣减，提供用量账单导出。
4. **图生视频等真实厂商接入**：补完混元 TC3 签名、可灵/即梦 SDK 调用与异步任务轮询回调。
5. **可观测性**：参考 Langfuse 思路接入调用链追踪与成本统计（已在诊断页预留扩展位）。
