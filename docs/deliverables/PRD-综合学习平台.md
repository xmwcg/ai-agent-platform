# 综合学习平台 — 产品需求与技术规划文档

> 版本：v1.0
> 状态：规划稿
> 目标：从 0 到 1 构建一个集成知识管理、AI 学习、内容创作和商业化的综合平台

---

## 一、产品定位与愿景

### 1.1 一句话定位
**面向开发者与技术学习者的全栈 AI 学习平台**：融合知识管理、互动课程、AI 对话、内容创作和商业化能力的一站式平台。

### 1.2 核心价值主张
| 用户角色 | 核心价值 |
|---------|---------|
| 个人学习者 | 个性化学习路径 + AI 辅助理解 + 随堂测验验证 |
| 内容创作者 | 工作流驱动的创作工具 + 知识变现 |
| 企业团队 | 私有化部署 + RBAC + 品牌定制 |

### 1.3 产品原则
1. **AI Native** —— 从第一天起就围绕 AI 能力设计，而非事后添加
2. **模块化** —— 每个功能可独立启用/关闭，支持渐进式采用
3. **可商业化** —— 每个 Phase 结束时都有可收费的功能
4. **体验优先** —— 课程和测验的体验质量 > 功能数量

## 三、核心功能详细说明

### 3.1 知识中枢（Knowledge Hub）

**能力**：
- Markdown 编辑器（Tiptap + 实时预览）
- 分类标签 + AI 自动打标签（基于内容语义）
- 全文搜索（Elasticsearch / Meilisearch）
- 版本历史与协同编辑（基础版）
- 知识文档与课程/测验双向关联

**关键设计决策**：
- 存储：Markdown 源文件 + 渲染后 HTML 双写
- 搜索：Meilisearch（轻量，开箱即用）vs ES（大规模）
- AI 标签：使用 chat 模型对文档摘要后生成标签，后台异步

### 3.2 学习中心（Learning Center）

**能力**：
- 课程结构化（章节 → 小节 → 内容块）
- 多格式内容：Markdown 文档、视频嵌入、交互代码块
- 随堂测验（5 种题型：单选/多选/判断/填空/代码）
- 学习路径 AI 推荐（依赖测验数据）
- 进度追踪 + 复习提醒
- **P0 课程**：Linux 入门到精通、云计算架构师 2026、大模型部署实战

**关键设计决策**：
- 课程是"结构化文档"而非视频为主——更适合 AI 交互和测验生成
- 测验题目严格基于课程内容生成（非通用题库）
- 学习路径使用 LangGraph 工作流：评估水平 → 检索课程 → 生成路径 → 优化时间

### 3.3 AI 对话与 Agent 系统

**能力**：
- 类 Codex 对话框界面
- 多 Provider 支持（OpenAI / Anthropic / Google / DeepSeek / 混元 / 通义千问 / 零一万物 / 月之暗面）
- 用户自定义 BaseURL + APIKey
- 对话历史管理 + 上下文窗口管理
- **代码解释**：粘贴代码 → 逐行解读 → 关联知识库
- **对比分析**：输入模型/工具名 → 结构化对比表
- RAG 模式：对话时自动检索知识库增强回答

**关键设计决策**：
- Agent 引擎抽象：`AgentProvider` 接口 → 各 Provider 实现
- MCP 集成：Agent 可调用外部工具（搜索、代码执行、图片生成）
- 对话上下文按 token 预算裁剪，保留关键历史

### 3.4 创作工坊（Creation Workshop）

**能力**：
- **文生图/图生图**：混元 Image + DALL-E 3 + Stable Diffusion
- **文生视频**：可灵 + Runway Gen-3
- **文档生成工作流**：可视化拖拽工作流引擎（节点：输入 → 处理 → 输出）
- **短视频工作流**：脚本 → AI 配音 → 字幕生成 → 视频合成

**关键设计决策**：
- 媒体生成 API 封装在统一 `MediaProvider` 接口后
- 生成任务异步队列（消息队列 + 轮询结果）
- 工作流引擎参考 n8n / LangFlow 架构但更轻量

### 3.5 商业系统

**能力**：
- 微信支付（JSAPI + Native + H5）
- 4 级会员体系：免费 / 基础 / Pro / 企业
- 内容付费：单课购买 / 专栏 / 会员全解锁
- AI 额度包：按 token 计费 / 按次计费 / 月包
- 订单管理与财务看板

**关键设计决策**：
- 额度系统独立于会员系统，支持"有会员 + 买额度"叠加
- 微信支付回调幂等处理

### 3.6 实践沙盒（Sandbox）

**能力**：
- **Phase 1**：Web Terminal（xterm.js + WebSocket PTY）—— Linux 命令练习
- **Phase 2**：Docker 隔离沙盒（Python / Node.js / Go 多语言运行环境）
- 30 分钟无操作自动销毁
- 预置课程配套练习环境

**安全设计**：
- 容器级别隔离
- 网络限制（只能访问白名单地址）
- 文件系统只读（除工作目录）
- CPU/内存配额硬限制

### 3.7 资讯聚合系统

**能力**：
- RSS 订阅管理
- 定时爬虫（每天凌晨抓取 AI/云厂商博客）
- AI 摘要提取（模型名、特性、Benchmark 自动提取）
- 推送通知（站内 + 邮件订阅）
- **模型发布日历**：月视图 + 厂商筛选 + 订阅推送

### 3.8 管理后台

**能力**：
- 用户管理（列表 / 禁用 / 角色分配）
- 内容管理（课程 / 文档 / 资讯审核）
- 订单管理
- 数据分析看板（DAU / 付费转化 / 课程完成率）
- 爬虫与资讯管理
- 系统配置（Provider 配置 / 额度定价 / 品牌设置）

### 5.2 技术选型建议

| 层级 | 技术选择 | 备选 | 选择理由 |
|------|---------|------|---------|
| **前端框架** | React 19 + Vite | Next.js | SPA 足够，Vite 构建快，Tauri 兼容性好 |
| **UI 组件库** | Ant Design 5 | Shadcn/ui | 成熟、企业级组件全、中文文档好 |
| **编辑器** | Tiptap（富文本）+ Monaco（代码） | Slate, CodeMirror | Tiptap 基于 ProseMirror 可扩展，Monaco 是 VS Code 内核 |

| **路由** | React Router v7 | TanStack Router | 生态成熟 |
| **可视化** | D3.js + ECharts | Apache ECharts | ECharts 开箱即用，D3 做知识图谱 |
| **终端** | xterm.js + @xterm/addon-fit | - | 标准选择 |
| **后端** | CloudBase（Node.js）+ Go（高性能服务） | Python FastAPI | CloudBase 降低运维，Go 做 AI Agent 网关和沙盒 |
| **主数据库** | PostgreSQL 16 | MySQL 8 | 向量扩展（pgvector）、JSONB、CTE |
| **向量数据库** | Milvus | Chroma, pgvector | 性能好、支持混合搜索 |
| **搜索** | Meilisearch | Elasticsearch | 轻量、开箱即用、中文分词好 |
| **缓存** | Redis 7 | - | 缓存 + 消息队列两用 |
| **对象存储** | MinIO | CloudBase 存储 | 兼容 S3 API，可私有部署 |
| **消息队列** | RabbitMQ | Redis Streams, Bull | 可靠、延迟任务支持好 |
| **CI/CD** | GitHub Actions | - | 生态最大 |
| **桌面端** | Tauri 2.0 | Electron | 包体积小、性能好、Rust 安全 |
| **AI Agent** | LangGraph + LangChain.js | 纯自定义 | 工作流编排成熟 |

## 七、补充分析与建议

### 7.1 用户原始计划中的隐藏风险

| 风险 | 说明 | 缓解建议 |
|------|------|---------|
| **Phase 1 范围过大** | 知识库+AI+RAG+课程+支付同时做，团队容易分散 | 严格定义 MVP Core，先砍到最少功能上线 |
| **AI Provider 绑定** | 依赖单一 API 提供商 | 从第一天起使用统一 Provider 接口 |
| **微信支付审核** | 需要企业资质和审核周期 | Phase 1 先做支付宝沙箱或虚拟支付，微信支付后续上线 |
| **RAG 质量** | 文档少时 RAG 效果差 | 预置一批高质量文档再启动 RAG 功能 |
| **测验质量** | AI 出题可能不准确 | 人工审核机制 + 用户反馈评分 |

### 7.3 商业价值增强建议
