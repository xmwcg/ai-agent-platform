# AI Agent Platform · 竞品对标与商业化优化方案

> 目标：对照业界主流 AI Agent / LLMOps 平台，校准本平台的功能深度与商业闭环，
> 确保「功能真实可用、付费可用、免费可用、对话真实」，而非 DEMO / 假功能。
> 日期：2026-07-19

---

## 一、指定 2 个竞品对标对象

| 竞品 | 定位 | 对标选取理由 |
|------|------|--------------|
| **Dify** | 开源 LLMOps / AI 应用开发平台（工作流编排 + RAG + Agent + 模型接入） | 工作流编排、RAG、模型接入广度、API 开放是最直接的对标维度；其商业化 SaaS 也是本平台的销售对标 |
| **Coze（扣子）** | 字节跳动 AI Bot 开发平台（插件/技能市场 + 多模态 + 对话体验 + 分发） | 「技能/插件市场 +  Bot 编排 + 多模态生产」是核心差异点；其对话体验与分发能力是用户体验对标基准 |

> 说明：二者均偏「开发者/创作者工具」，而本平台额外具备 **微信支付 v3 真实收款、积分/分销商业闭环、企业私有化部署（金网通/中转站）**，这是相对竞品的商业化纵深优势，应作为销售卖点保留并强化。

## 三、本轮已完成的真实化改造（不动框架，仅补全实现）

1. **工作流 skill 节点（真实执行）**
   - `server/src/services/workflow-engine.service.ts`：skill 节点从「占位报错」改为真实调用
     `getSkill(skillId)`（内置技能）或 `UserSkill`（用户技能：prompt / mcp / workflow 三类），
     成功返回真实输出，失败返回明确错误（绝不伪装成功）。
   - 编辑器 `WorkflowEditor.tsx` 新增技能下拉选择器，可绑定技能并被执行。
2. **仪表盘 5xx / 延迟（真实数据）**
   - `middleware/apm.ts`：新增 5xx 时间戳采样与最近请求耗时样本，导出 `getToday5xxCount()` / `getLatencyPercentiles()` / `getSuccessRatePercent()`。
   - `services/dashboard.service.ts`：运营仪表盘 `today5xx / p95Ms / p99Ms / uptimePercent` 由常量 0 改为真实 APM 数据。
3. **RAG 增量嵌入（真实）**
   - `services/rag.ts`：`embedNewDocuments()` 实现为「嵌入所有缺失向量的文档」，支持 `sinceMs` 增量窗口。
   - `routes/knowledge.ts`：新建知识文档后异步自动增量嵌入，新文档立即可被检索。
4. **修改密码（真实）**
   - `client/src/services/api.ts` + `ProfilePage.tsx`：接入后端 `PUT /api/auth/change-password`，含长度校验与一致性校验，修改后强制其他设备登出。
5. **短视频工作流（真实可用）**
   - `CreativeWorkshop.tsx`：卡片由「即将上线」改为「已就绪」并绑定路由。
   - 新增 `client/src/pages/VideoWorkflow.tsx`：调用真实 `video-pipeline` 技能，输出调研 + 口播脚本/分镜，支持下载方案。
