# 方案②：轻量服务器 docker-compose 一键起全栈（★ 推荐主线）

> 适用前提（用户已确认）：**公司主体 + 已有公司微信支付商户号 + 可购买轻量服务器 + 目标市场国内为主 + 要求稳定/专业/闭环**。
> 闭环定义：**开发 → 部署 → 上线 → 运营 → 免费引流获客 → 付费 → 再获客 → 再付费**。
> 本方案对现有代码**零改造**即可真收款，是落地全闭环成本最低、风险最小的主路径。

---

## 0. 本方案一句话结论

买一台国内轻量服务器（≈¥50/月），`docker-compose up -d` 把已有全栈一次性拉起，配好域名 ICP 备案 + 微信支付商户号联调，即可打通「用户订阅 → 微信收款 → 激活会员」完整商业化闭环。AI 调用用 DeepSeek/混元免费额度起步，单用户边际成本极低。

## 1. 项目定位 / 能做什么 / 解决哪些痛点 / 特色功能

### 1.1 项目类别
**一站式 AI 学习 / 创作 / 生产力 + 商业变现 SaaS 平台**，对标 Dify / Coze / FastGPT / LibreChat，但定位更偏「**中小企业与个人创作者的国产大模型驱动工作台 + 可对外收费的开放能力平台**」。

技术事实（已查证 `README.md`、`docker-compose.yml`、`server/.env.example`）：
- 前端 React18 + TypeScript + Vite + Ant Design 5（`client/`）
- 后端 Express + TypeScript + Mongoose(MongoDB) + Redis(ioredis)（`server/`）
- 认证 JWT + bcrypt；扩展 MCP 真实 SDK；支付抽象层（微信 v3 / Stripe / Mock）

### 1.2 已落地能力（真实可用，非占位）
| 模块 | 现状 |

| AI 对话 | 多 Provider、多会话、Redis 持久化（`/api/ai`） |

> 完整 API 与模块清单见 `README.md` 与 `docs/API.md`，本文不重复罗列。

### 1.3 解决的痛点（对应 `docs/PAIN-POINTS.md` 六大痛点）
1. **部署/配置复杂** → `GET /api/diagnostics` 一键自检 + 前端部署自检页，Redis 缺失自动降级内存，0 依赖可跑。
2. **AI 客服黑盒不可信** → RAG 答案返回**来源引用（docId/标题/置信度）+ 触发词转人工 + 满意度评分**闭环。
3. **上手门槛高** → 场景化快速启动模板（客服/新媒体/备考/研发），一键生成「知识文档+机器人」。
4. **多模型接入割裂** → 大模型配置中心运行时自助接入 OpenAI/DeepSeek/混元/通义/智谱，全平台统一驱动。
5. **媒体厂商分散** → 媒体生成 Provider 抽象（混元/可灵/即梦/Mock）统一接口，配置即切换。
6. **企业协作与开放变现缺位** → 团队 RBAC + 开放 API 市场按量计费（API Key 签发、日配额计量、超阈 429）。

### 1.4 特色功能（国内竞品普遍薄弱、本项目已具备）
- **可追溯可信客服闭环**（来源引用 + 转人工 + 满意度）—— FastGPT 薄弱环节的补强。
- **部署自检看板**——降低安装痛点的硬科技。
- **场景化一键启动模板**——对标模板市场但聚焦「知识库+客服」组合。
- **开放 API 市场按量计费**——把能力变成可收费开放平台。
- **媒体生成多厂商抽象 + 腾讯云 TC3 真实验签**（与对话复用同一签名算法）。

## 2. 国内竞品对比 & 差异化 / 可扩展垂直痛点

### 2.1 竞品定位速览（2026 联网核对）
| 竞品 | 强项 | 弱项/空白 | 本项目可切入 |
|------|------|-----------|--------------|
| **Coze（扣子）** | 零代码、上手最快、字节生态 | 私有化弱、能力不可自托管、难对外收费 | 自托管 + 开放 API 市场按量收费 |
| **Dify** | 开源、深度定制、私有化 | 商业变现/支付需自建、客服可信性弱 | 内置套餐/配额/支付闭环 + 可信客服 |
| **FastGPT** | 专精 RAG/知识库客服 | 答案不可追溯、缺团队 RBAC、缺开放计费 | 来源引用 + 转人工 + 满意度 + 团队隔离 |
| **百度千帆 / 阿里百炼** | 模型供给 + 企业级 | 偏 PaaS、缺开箱即用工作台与前端 | 直接可用前端 + 创作/学习矩阵 |
| **LibreChat** | 多模型对话 UI | 无商业变现、无客服/团队 | 全套变现 + 团队 RBAC |

> 结论：**本项目不需要与巨头正面拼"模型能力"，而是拼"国产化开箱即用 + 可信客服 + 内置变现闭环 + 可对外开放收费"这个组合**——这是 Dify/Coze/FastGPT 都没做满的空白带。

### 2.2 国内高频痛点 & 无人/少人解决的垂直细分（★ 放入后续优化开发方案）
1. **「可信 AI 客服」在监管与责任场景下是刚需**：金融/医疗/政务类客服**必须能给出答案依据与责任人**，目前市面多为黑盒。→ 强化「来源引用 + 审计日志 + 人工兜底」形成合规客服子产品。
2. **县域/传统行业（工厂、律所、培训机构、诊所）的"零技术接入"**：它们有知识库需求但无技术团队。→ 快速启动模板 + 行业预置知识包（如「诊所 FAQ」「律所合同问答」）做成垂直 SaaS 模板市场。
3. **中小微企业"AI 能力对外收费"通道缺失**：很多公司想把自己训练的客服/翻译能力打包卖给客户，但缺计量与计费。→ 开放 API 市场 + 渠道分润（见 `docs/BUSINESS-MODEL.md` 途径 6）。
4. **内容生产者"一条龙"缺位**：写文案→配图→配音→成片分散在不同工具。→ 视频生产流水线（`video-pipeline` + MoneyPrinterTurbo）补齐，做成「自媒体矩阵」垂直方案。
5. **教育/考证垂直**：考试备考场景的「知识库+RAG 答疑+学习路径」组合，结合课程系统，可做考证垂类。

> 上述 1–5 作为 Phase 4/5 与垂直版路线图，详见第 8 章「后续优化开发方案」。

## 4. 门槛打通清单（支付 / 备案 / 质量 / 安全）

### 4.1 支付（微信支付商户号联调）—— 用户已有，需做"配置 + 验签联调"
复用已有 `services/payment.service.ts`（已实现 `WechatGateway`：统一下单 + 回调验签 AES-256-GCM 解密 + RSA 验签）。

**步骤清单（可勾选）：**
- [ ] 在 `server/.env` 填入微信商户号参数（对照 `server/.env.example`）：
  - `WECHAT_MCH_ID`、`WECHAT_APP_ID`、`WECHAT_API_V3_KEY`、`WECHAT_CERT_SERIAL`、`WECHAT_PRIVATE_KEY`、`WECHAT_NOTIFY_URL`
- [ ] `WECHAT_NOTIFY_URL` 设为公网可访问的 `https://你的域名/api/billing/webhook/wechat`（需 HTTPS，见 4.4）
- [ ] `DEFAULT_PAY_PROVIDER=wechat`，`ENABLE_MOCK_MODE=false`
- [ ] **端到端联调验收**（关键质量门）：
  - [ ] 走「下单 `POST /api/billing/orders` → 微信扫码/支付 → 回调 `webhook/wechat` → 验签通过 → 会员 `plan` 升级、`membershipExpiresAt` 写入」
  - [ ] 验证 `resolveUserPlan` 过期自动降级 free（见 `docs/BUSINESS-MODEL.md` 合规要点）
  - [ ] 用沙箱/小额（¥0.01）真实支付跑通一次，确认回调验签不丢单
- [ ] 证书公钥（`WECHAT_PLATFORM_CERT`）配置到位，防伪造订单（生产必备）

### 4.2 审批 / 备案（国内上线硬门槛）
- [ ] **ICP 备案**：域名 + 国内服务器，腾讯云/阿里云控制台免费提交，约 1–2 周。*未备案域名国内无法解析到国内服务器。*
- [ ] **经营性 ICP 许可证（ICP 证）**：若平台"有偿信息服务"且营收达阈值，需办（¥数千 + 周期 1–3 月）。早期可先用备案号运营、视规模补办；**建议咨询当地通管局/代办**。
- [ ] **增值电信（EDI）**：涉及"在线数据处理与交易处理"时可能需办。同上为合规进阶项，早期可后置。
- [ ] **支付合规**：商户号主体需与备案主体/签约主体一致；用户已有公司商户号，确保平台运营主体一致即可。

## 6. 国内主流模型 API 接入 & 第三方模型接入（★ 用户重点诉求）

项目已通过 `gateway/ai-gateway.service.ts`（OmniRoute 式统一路由：provider 注册表 + 前缀寻址 + priority fallback）与 `/api/model-config`（运行时自助接入）原生支持。无需改代码，仅改 `.env`。

### 6.1 零成本起步（Mock 模式）
```
ENABLE_MOCK_MODE=true
DEFAULT_AI_PROVIDER=mock
```
→ 0 Key 可跑通全站演示，先验证产品与获客，再接真模型。

### 6.2 接入国内主流模型（推荐生产起步，成本最低）
在 `server/.env` 配置（对照 `server/.env.example`）：
| 模型 | 变量 | 备注 |
|------|------|------|
| **DeepSeek** | `DEEPSEEK_API_KEY=` | 国产最便宜之一，对话/代码首选 |
| **腾讯混元** | `HUNYUAN_SECRET_ID=` / `HUNYUAN_SECRET_KEY=` | 同时驱动对话/文生图/媒体生成，TC3 真实验签已落地 |
| **通义千问** | `QWEN_API_KEY=` | 阿里系模型 |
| **智谱 GLM** | 经 `/api/model-config` 自助接入 | 自定义 OpenAI 兼容端点 |
- `DEFAULT_AI_PROVIDER=deepseek`（或 hunyuan）让默认对话走国产模型。

### 6.3 接入第三方 / 海外模型
| 模型 | 变量 | 用途 |
|------|------|------|
| OpenAI | `OPENAI_API_KEY=` / `OPENAI_BASE_URL=` | GPT 系列 |
| Anthropic | `ANTHROPIC_API_KEY=` | Claude 系列 |
| 自定义兼容端点 | `/api/model-config` 填 baseURL+key | 任何 OpenAI 兼容服务 |

> **自助接入步骤**（前端 `/model-config`）：登录 → 模型配置中心 → 新增 provider（填名称/baseURL/key/模型名）→ 测试连接 → 设为默认。全平台对话/RAG/客服/生成即换模型，零改代码。

### 6.4 成本结构（AI 调用）
- 国产模型（DeepSeek/混元）对话成本约 ¥1/百万 token 级，远低于 OpenAI 5–10 倍。
- 免费版配额限流（对话 20/日等，见 `docs/BUSINESS-MODEL.md`）天然控制成本；付费用户 ARPU（¥39/¥99）远超模型成本，**单位经济为正**。

## 10. 风险与回退
- **服务器宕机**：docker-compose `restart: unless-stopped` + 监控 + 备份；可随时迁移镜像到另一台服务器（方案② 最大优势：可移植）。
- **备案延迟**：先用服务器 IP/临时域名自测，备案期间不对外推广。
- **支付纠纷**：订单历史 + 取消订阅 + 验签防伪造，满足消费者权益。

## 附：与方案①的核心差异（详见 `PLAN-A-Serverless.md`）
- 方案②：**零改造、最稳、可直接真收款**，新增现金仅服务器月费 ≈¥35–75。
- 方案①：成本更低（≈¥0–30/月），但 MCP stdio / 媒体异步 / Redis 需 Serverless 适配改造，且国内用 EdgeOne+国内 MongoDB。
- 两方案"支付/订阅闭环"能力一致（复用同一支付抽象层），差异仅在后端运行形态。
