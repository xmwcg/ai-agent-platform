## 1. 用户与项目总目标

用户拥有并运营 `aibak.site`，主项目为 AIbak / Reasonix AI 全栈 AI 应用平台。用户要的不是演示站或模块集合，而是：

> 一个完整可商用、可收费、可销售、可自动交付、可售后、可持续运营的正式平台网站。

所有模块必须形成真实业务闭环：

```text
用户发现功能
→ 理解价值
→ 注册/登录
→ 配置或选择模型
→ 实际完成任务
→ 保存/复制/分享/下载结果
→ 使用额度或付费
→ 权益自动到账
→ 获取产品或服务
→ 售后、退款、发票、客服
→ 管理后台可审计和对账
```

禁止以下“伪完成”定义：

- 页面存在但接口不可用；
- 按钮存在但404、空跳转或只有提示；
- 接口存在但生产未配置；
- Mock结果冒充真实AI结果；
- 支付按钮存在但支付、回调、权益、交付未打通；
- 本地测试通过就宣称生产完成；
- 只修用户指出的一个问题，不检查相邻模块；
- 以“代码已经写了”代替实际验收。

- 真实模型调用；
- 90秒以上合理超时；
- 可取消；
- 错误原因可见；
- 会话历史；

- 输出支持复制、重新生成、导出、分享、下载。
