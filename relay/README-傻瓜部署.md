# 中转站 傻瓜部署清单（复制粘贴就能跑）

> 这套东西独立运行，不影响你现有的 aibak.site 官网。全程照抄命令即可。

## 你需要先准备
1. 一台有公网 IP 的云服务器（你已经有）。
2. 一个网址，比如 `relay.aibak.site`。
3. 各家大模型的"钥匙"（API Key），比如硅基流动、DeepSeek 官方等（后面在网页里填，现在不用管）。

---

## 第 1 步：登上你的服务器
用服务商网页上的"远程连接/终端"按钮，或电脑上的 SSH 工具进去。看到能敲命令的黑框就算成功。

## 第 2 步：装"搬运工" Docker（只需装一次）
把下面这行**整行复制**进去，回车，等它跑完：
```
curl -fsSL https://get.docker.com | sh
```

## 第 3 步：把中转站文件弄到服务器
把下面**整段**复制进去回车（它会自动下载仓库并进入中转站文件夹）：
```
git clone https://cnb.cool/aibak.site/ai-agent-platform.git && cd ai-agent-platform/relay
```

## 第 4 步：改成你自己的网址
把 `Caddyfile` 里的 `relay.aibak.site` 换成你要用的网址。懒得用编辑器的话，直接抄这行（把两个 relay.aibak.site 换成你的）：
```
sed -i 's/relay.aibak.site/你的网址/g' Caddyfile
```
> 例：`sed -i 's/relay.aibak.site/ai.aibak.site/g' Caddyfile`

## 第 5 步：去域名后台，把网址指向服务器
到你买域名的网站后台，加一条解析记录：
- 类型：`A`
- 主机记录：`relay`（如果你用 relay.aibak.site）
- 记录值：填你**服务器的公网 IP**

> 大白话：就是告诉全世界"relay.aibak.site 这个门牌，对应我这台服务器"。

## 第 6 步：开机！
```
docker compose up -d
```
等一会儿（第一次要下载程序，慢一点正常）。

## 第 7 步：进门设置
浏览器打开 `https://你的网址`：
1. 用初始管理员账号登录，**马上改密码**。
2. 「渠道管理」→ 逐个把各家大模型的"钥匙"填进去。
3. 「令牌管理」→ 生成一把给金网通用的"钥匙"（token），复制下来。

## 第 8 步：金网通接上
在金网通控制台 **⑥ 大模型接入配置**：
- 厂商选「我的中转站·智控版」
- 网址填 `https://你的网址/v1`
- 钥匙填第 7 步复制的那把 token
- 点「获取模型列表」「测试连接」，通了就成功。

---

## ⚠️ 一个重要提醒（关于端口）
如果这台服务器**已经在跑你的 aibak.site 官网**（官网占用了 80 和 443 端口），那第 6 步会报错"端口被占用"。

两个办法二选一：
- **办法A（最省事）**：把中转站装到**另一台服务器**上，各跑各的。
- **办法B（同一台共用）**：告诉我"要和官网共用一台"，我把配置改成走你现有官网的反向代理（不抢 80/443）。

不确定属于哪种？直接把第 6 步的报错发我，我帮你判断。
