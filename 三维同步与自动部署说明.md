# 四端同步与自动部署说明（兼容入口）

当前生产发布的唯一有效方案已经统一到：

- 本地只推 CNB `main`。
- CNB 流水线先执行后端测试、类型检查和前端构建。
- 门禁通过后，安全镜像 GitHub，并晋级 `deploy/production`。
- 生产服务器的 systemd timer 只轮询 `deploy/production`，部署成功且公网验收通过后才记录版本。

完整安装、回滚和验收说明见：[`docs/CNB-FOUR-END-SYNC.md`](docs/CNB-FOUR-END-SYNC.md)。

## 兼容文件

历史上的 `deploy-from-cnb.sh` 和 `webhook-server.py` 仍保留为可选兼容入口，但它们**不能直接部署 `main`**，只会唤醒安全 watcher。生产主通道仍是服务器主动拉取，不要求开放入站端口。

## 禁止事项

- 不要直接在 GitHub 或生产服务器修改正式代码。
- 不要恢复 GitHub → CNB 的定时反向同步。
- 不要让 webhook 或 cron 直接部署 CNB `main`。
- 不要使用无条件 `git push --force`。
