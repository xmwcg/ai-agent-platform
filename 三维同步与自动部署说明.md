# 三维同步与自动部署操作说明

> 适用项目：ai-agent-platform（AIBAK 前端 + 后端 monorepo）
> 目标：让你在任何地方改代码，都能自动、安全地部署上线，并有云端备份。

---

## 一、什么是"三维同步"

你的代码在三个地方保持一致：

| 维度 | 是什么 | 作用 |
|------|--------|------|
| 本地电脑 | 你写代码的地方 | 日常开发 |
| 服务器 | `159.75.124.59` 上运行的项目 | 真正给用户访问、能收钱的地方（已上线） |
| GitHub | `github.com/xmwcg/ai-agent-platform` | 云端备份仓库 |

---

## 二、两条自动线

### 线 A：本地 → 服务器 → GitHub（你日常开发，全自动）

```
你本地改代码
   ↓ 自动部署脚本 scripts/auto-deploy.cjs（防抖 30s）
推到服务器（先上线）
   ↓ 服务器钩子 post-receive
① 自动构建部署（后台，日志 /var/log/ai-platform-deploy.log）
② 自动把代码镜像备份到 GitHub
```

你只管写代码、推一次，剩下的全自动。

### 线 B：GitHub → 服务器（你不在电脑旁时）

```
手机/电脑在 GitHub 网页改代码（或本地 git push github）
   ↓ 触发 GitHub Actions（.github/workflows/deploy.yml）
自动把代码推到服务器并部署上线
```

适合：出门在外、电脑关着，掏手机就能改 bug / 改配置。

---

## 三、怎么用"线 B"（手机改 GitHub 自动部署）

1. 打开 `github.com/xmwcg/ai-agent-platform`
2. 找到要改的文件 → 点编辑（铅笔图标）→ 改完点 `Commit changes` 保存
   - 或在本地终端：`git push github main`
3. 等 2–5 分钟，服务器自动重新构建上线
4. 查看进度：
   - GitHub 仓库 → **Actions** 标签看部署机器人运行状态
   - 服务器：`ssh root@159.75.124.59` 后 `tail -f /var/log/ai-platform-deploy.log`

---

## 四、密钥与权限（已配置好，一般不用动）

- GitHub 仓库 **Settings → Secrets and variables → Actions** 里有：
  - `DEPLOY_SSH_KEY`：GitHub 推代码到服务器的专用私钥（只用于部署，不能进代码、不能外泄）
- 服务器上的敏感配置（数据库密码、支付密钥、大模型 Key 等）都放在服务器 `server/.env`，**不要**提交到 GitHub。

---

## 五、CI 测试（.github/workflows/ci.yml）

- 每次 push 都会跑：
  - 后端：`tsc` 类型检查 + 单元测试（`npm test`）
  - 前端：`tsc` 类型检查 + `lint`
- **测试只在 GitHub 云端跑，不会影响服务器上运行的项目。**
- 测试出问题会显示红色 ❌，但**不阻塞部署**（部署由单独的 deploy.yml 负责）。

---

## 六、防死循环（技术细节，可跳过）

GitHub → 服务器的推送使用的是一把带标记的专用部署钥匙；服务器收到后识别到"是机器人推来的"，就**不再把代码推回 GitHub**，从而避免无限循环。

---

## 七、常见问题

**Q：我在 GitHub 改了代码，但服务器没更新？**
A：先看 GitHub 仓库 **Actions** 里对应运行是不是红色失败；再看服务器部署日志。常见原因：密钥失效、代码类型检查没过。

**Q：本地连不上 GitHub 怎么办？**
A：用能连 GitHub 的机器（或服务器）推送；或直接用 GitHub 网页编辑，完全不需要本地能连。

**Q：CI 出现红色 ❌ 要紧吗？**
A：不阻塞上线，但说明代码有测试/类型问题，建议尽快修（测试只在云端跑，不碰服务器）。

**Q：哪些东西不能放 GitHub？**
A：密码、支付密钥、API Key、服务器 IP 白名单等敏感信息留在服务器 `server/.env`；代码和配置可以放。

---

## 八、手动触发部署

在 GitHub 仓库 **Actions → Deploy to Server → Run workflow** 可手动点按钮触发一次部署（无需改代码）。
