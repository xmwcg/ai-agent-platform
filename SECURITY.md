# 安全策略

## 支持版本

| 版本    | 支持状态     |
| ------- | ------------ |
| 1.x     | ✅ 积极支持  |
| < 1.0   | ❌ 不再支持  |

## 报告漏洞

请**不要**通过公开 Issue 报告安全漏洞。

请将漏洞详情发送至 **security@aibak.site**，我们将在 48 小时内响应。

### 漏洞报告应包含：
- 漏洞描述
- 复现步骤
- 影响范围
- 建议修复方案（可选）

## 安全措施

本项目实施了以下安全措施：

### 传输安全
- 所有通信强制 HTTPS
- HSTS 头已配置
- TLS 1.2+ 最低要求

### 认证与授权
- JWT 令牌认证（HS256，72 小时过期）
- 基于角色的访问控制（RBAC）
- API Key 哈希存储（SHA-256）
- 密码 bcrypt 哈希（salt rounds = 10）

### 数据保护
- 支付密钥仅存环境变量，不落盘
- 敏感字段加密存储
- API Key 明文仅首次返回
- 支付 Webhook HMAC 签名验证
- 重放攻击防护（5 分钟时间窗口）

### 输入验证
- 请求 body 大小限制 10MB
- 全局 API 频率限制
- 输入参数白名单校验
- SQL/NoSQL 注入防护（Mongoose ORM + 参数化查询）

### 依赖安全
- 定期 `npm audit` 扫描
- Dependabot 自动更新配置
- 关键依赖锁定版本

### 生产部署
- Helmet 安全头
- CORS 白名单
- 信任代理 X-Forwarded-* 头
- 容器非 root 运行
- 只读文件系统（容器内）

## 安全披露时间线

- 0-48h：确认收到报告
- 48h-7d：评估并确认漏洞
- 7d-30d：开发并测试修复
- 30d-45d：发布修复版本
- 45d+：公开披露（如适用）
